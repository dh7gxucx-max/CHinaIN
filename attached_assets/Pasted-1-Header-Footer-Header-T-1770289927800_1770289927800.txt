1. Глобальная навигация (Header & Footer)
Функционал доступен на любой странице.

Header (Шапка):

Ticker (Бегущая строка): Курс Юаня к Рупии | Статус склада (Гуанчжоу: Прием открыт).

SIP-Widget: Крупный номер телефона с индикатором «Очередь на линии: 0 чел» + кнопка «Заказать звонок» (Callback).

Меню: Главная | Тарифы | Магазины (Где купить) | Помощь.

Профиль: Кнопка «Личный кабинет» / «Регистрация».

Footer (Подвал):

Legal: Ссылка на публичную оферту (особенно пункт про обязательства COD).

Download App: Ссылки на APK (Android).

Contacts: Адрес офиса в Индии, WhatsApp Business API ссылка.

2. Главная страница (Public Front)
Задача: Объяснить схему «Закажи в Китае — Плати дома в Индии».

Hero-Section (Первый экран):

Заголовок: «Шоппинг в Китае без предоплаты за доставку».

Схема: Получи адрес склада -> Закажи товар -> Оплати доставку курьеру при вручении.

CTA (Кнопка): «Получить адрес в Китае бесплатно».

Блок доверия (SIP-акцент):

«Живая поддержка 24/7: Мы не прячемся за чат-ботами. Позвоните нам прямо сейчас».

Блок «Калькулятор»:

Ввод веса (кг) -> Расчет: [Фрахт + Таможня (10%) + Комиссия].

Итог: «Примерная сумма к оплате курьеру: ₹XXXX».

3. Личный кабинет (User Dashboard) — Ядро системы
Здесь происходит управление посылками и голосовая верификация.

А. Главная панель (Dashboard Home)
Виджет «Мой адрес в Китае»:

Поля: Имя (с уникальным ID клиента), Телефон склада, Адрес на китайском (кнопка «Копировать»), Адрес на английском.

Виджет «Trust Score» (Рейтинг надежности):

Шкала: от 0 до 100.

Лимит COD: «Вам доступен заказ с оплатой при получении на сумму до ₹15,000». (Растет с количеством успешных выкупов).

Б. Раздел «Входящие» (Inbound Parcels)
Кнопка «Добавить трек»: Пользователь вносит трек-номер от продавца (Taobao/Shein).

Список посылок на складе:

Статус: «Принято», «Взвешивание», «Проверка».

Фото-отчет: 3 фото товара (этикетка, общий вид).

Действие: Чекбокс «Готов к отправке».

В. Раздел «Оформление отправки» (Verification Gateway)
Это замена платежного шлюза. Самый сложный технический блок.

Сводка: Список выбранных посылок -> Общий вес.

Выбор адреса доставки: Добавление индийского адреса.

KYC Check: Загрузка Aadhaar Card (если не загружено ранее). Без этого кнопка дальше не работает.

Расчет COD: Полная сумма, которую клиент должен будет отдать курьеру.

Блок «Голосовой Контракт» (Voice Contract via SIP):

Текст: «Для отправки груза необходимо подтвердить намерение оплатить ₹2,500 при получении».

Кнопка: «Позвонить мне для подтверждения».

Логика: Система звонит клиенту -> Робот проговаривает сумму -> Клиент нажимает "1".

Результат: Статус меняется на «Контракт подписан голосом». Посылка уходит на упаковку.

Г. Раздел «Отслеживание» (Tracking)
Таймлайн: Китай -> Таможня -> Курьер в городе.

Статус «К оплате»: Напоминание красным цветом: «Приготовьте ₹2,500 наличными или UPI».

4. Раздел «Помощь и Коммуникация» (Support Hub)
Интерфейс управления SIP-обращениями.

Тикет-система: Создать текстовое обращение.

Заказ звонка (Callback):

Выбор темы: «Проблема с товаром», «Перенос доставки», «Вопросы по оплате».

Выбор времени: «Позвонить сейчас» или «В интервале 14:00-16:00».

История звонков: Лог всех разговоров с менеджерами (для прозрачности).

5. Раздел «Каталог магазинов» (Directory)
Помощь в выборе товаров, так как вы сами не продаете.

Ссылки на проверенные магазины на Taobao/1688 по категориям (Одежда, Электроника, Детское).

Инструкции: «Как зарегистрироваться на Taobao из Индии».

6. Админ-панель (Back-office для персонала)
Скрытая часть сайта для управления бизнесом.

Модуль «Верификация и Риски»
Список COD-заказов:

Фильтр по статусу звонка (Успешно / Сброс / Не дозвонился).

Возможность прослушать запись подтверждения (доказательство для споров).

Blacklist: Ручное или автоматическое снижение Trust Score за отказ от посылки.

Модуль «Склад и Логистика»
Приемка треков (сканер штрих-кодов).

Загрузка фото.

Генерация накладных для «Последней мили» (интеграция с Delhivery/Ecom Express).

Модуль «Финансы (COD Reconciliation)»
Сверка: [Ожидаемая сумма COD] vs [Фактически полученные деньги от курьерской службы].

Выявление "потерянных" денег.

Ключевая особенность (User Flow)
Весь сайт построен так, чтобы пользователь не мог отправить посылку, пока система не убедится (через SIP-звонок и KYC документы), что это реальный человек, готовый платить. Это фильтр, который делает бизнес на COD рентабельным.
